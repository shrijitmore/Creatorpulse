# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| main branch | Yes |
| older branches | No |

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Email: orangeglazz@gmail.com

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Your suggested fix (optional)

You will receive a response within 48 hours. If the issue is confirmed, a fix will be released as soon as possible.

## Scope

In scope:
- Authentication bypass
- Injection vulnerabilities (SQL, command, prompt)
- Exposed API keys or secrets
- Unauthorized data access
- CSRF / XSS

Out of scope:
- Denial of service
- Social engineering
- Issues in third-party dependencies (report to them directly)

## Disclosure Policy

We follow responsible disclosure. After a fix is released, we will credit the reporter (unless they prefer to remain anonymous).
